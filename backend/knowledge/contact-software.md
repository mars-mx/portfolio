Marius war von 2019 bis 2023 als Werkstudent bei der CONTACT Software GmbH,
einem führenden Anbieter von Software für Product Lifecycle Management (PLM),
neben dem Studium in der Softwareentwicklung tätig. Er arbeitete im Team des
CONTACT Portals an produktnaher Software, überwiegend in Python.

## Das CONTACT Portal

Marius hat das CONTACT Portal mit aufgebaut: die erste vollständig
webbasierte, voll funktionsfähige Umgebung auf Basis von CONTACT Elements.
Sie war zugleich die erste Anwendung von CONTACT, die produktiv in Kubernetes
lief, damals auf AWS. Er war unter anderem an den Software Downloads, dem
Ticketsystem, den Editorials (redaktionelle Inhalte) und den
Softwareumgebungen beteiligt.

## Customizing auf CONTACT Elements

Vom Charakter her war das CONTACT Portal ein Customizing auf einer Instanz
von CONTACT Elements: bewusst sauber und ohne Wildwuchs, immer im Rahmen
dessen, was das Produkt an Customizing vorsieht, und zugleich als Demo für
die Webfähigkeit von CONTACT Elements gedacht. Die besonderen
Herausforderungen dabei: Multitenancy über organisatorische Kontexte und ein
konsequent webbasierter Ansatz (Web first).

## Zusammenarbeit mit Produktmanagement und UX

Beim Aufbau des CONTACT Portals lag Marius' Fokus auf enger Abstimmung mit
Produktmanagement und UX: eine konsistente Corporate Identity sicherstellen
und gleichzeitig Anforderungen aus Kundensicht bzw. User Stories aus dem
Produktmanagement in tatsächliche Features einer modernen Webanwendung
übersetzen.

## Betrieb in Kubernetes und SaaS

Marius arbeitete eng mit dem Cloud Ops Team zusammen und war unmittelbar
dabei, als CONTACT die ersten SaaS-Kunden live brachte. Aus dieser Zeit kennt
er den Betrieb von Enterprise Software in Kubernetes aus der Praxis.

## Technologien

- Python (Customizing von CONTACT Elements, Backend)
- React (Single Page Application im Frontend)
- Kubernetes und AWS (produktiver Betrieb)
- Terraform (Infrastructure as Code im Enterprise-Bereich)
- Ansible Vault (Secret Management)
- GitLab CI (Continuous Integration & Deployment)

## Als Freelancer im Ökosystem von CONTACT

Auch nach seiner Zeit bei CONTACT Software arbeitet Marius im Ökosystem von
CONTACT: Seit 2024 migriert er als Freelancer für einen Kunden aus der
Chemiebranche einen Legacy CORBA Client des PLM-Systems von CONTACT Software
in eine moderne React SPA und customized die bestehenden Schnittstellen
(Python, Morepath). Marius kennt CONTACT Elements damit von beiden Seiten:
aus der Produktentwicklung beim Anbieter und aus dem Customizing auf
Kundenseite.
